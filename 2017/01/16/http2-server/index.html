发表于 2017-01-16

搭建server端

使用node.js来搭建http2.0的server环境，
1，下载nodejs的http2.0模块，具体配置可参考https://github.com/molnarg/node-http2

2，当构建好之后，需要升级我们的nodejs版本，至少到5.0才能够支持http2.0协议

3，这时我们就可以创建自己的server代码了，

4，因为http2.0协议需要走 https（可支持不走https请求），我们需要创建自己的证书，可以直接在线创建免费的证书免费证书创建
创建完成后，添加到项目目录下即可，

5，在浏览器中打开：
https://localhost:8080
如果看到返回的数据，即server搭建成功

了解https

因为HTTP2.0协议，是在SSL层上进行的，所以了解这个必须要先去了解下https工作原理，

  • 浏览器向服务器请求时，会带上SSL协议版本号，加密算法总类，以及通信需要的各种信息等
  • 服务器像客户端发送自己的SSL协议版本号，加密算法等其他信息，同时会带上自己的证书
  • 客户端开始验证证书的合法性，主要验证有证书是否过期，发行服务器证书的CA 是否可靠，发行者证书的公钥能否正确解开服务器证书的“发行者的数字签名”，服务器证书上的域名是否和服务器的实际域名相匹配
  • 验证通过，生成一个后面数据通信的对称秘钥，在用server的公钥进行加密，传给server
  • server用非对称的私钥来进行解密，获取到这个对称公钥，同时在SSL通讯过程中还要完成数据通讯的完整性，防止数据通讯中的任何变化
  • 客户端向服务器端发出信息，指明后面的数据通讯将使用的步骤⑤中的主密码为对称密钥，同时通知服务器客户端的握手过程结束
  • 服务器向客户端发出信息，指明后面的数据通讯将使用的步骤⑤中的主密码为对称密钥，同时通知客户端服务器端的握手过程结束
  • SSL 的握手部分结束，SSL 安全通道的数据通讯开始，客户和服务器开始使用相同的对称密钥进行数据通讯，同时进行通讯完整性的检验
